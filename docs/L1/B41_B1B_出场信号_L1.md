# B41/B1B 出场信号 — 设计文档 L1

## 1. 需求

B41/B1B 买入信号触发后，以 `cross(75, RSI)` 作为出场信号。S41/S1B 卖出信号触发后，以 `cross(RSI, TH)` 作为出场信号。

约束：
- 有信号才算，无活跃持仓不额外计算
- 出场信号推送完成后标记已出场
- 出场信号宁多勿少（允许去重，不允许丢失）
- 支持 1m / 5m / 30m / 1d 周期，区分股票
- 便于未来扩展新函数的出场规则

---

## 2. 架构：双路径兜底

```
┌─────────────────────────────────────────────────────────┐
│                      每次 tick                          │
│                                                         │
│  主路径（Engine tracker）         兜底路径（calculate 内）│
│  ┌─────────────────────┐       ┌──────────────────────┐ │
│  │ 查 TB tracker       │       │ _compute 返回 rsih   │ │
│  │ suppress 已有持仓信号│       │ 扫 B41/B1B/S41/S1B   │ │
│  │ 写库 + 推送         │       │ 找对应 cross bar      │ │
│  └─────────────────────┘       │ 生成出场信号          │ │
│          │                      └──────────┬───────────┘ │
│          │         write_signals 去重       │             │
│          └──────────────┬──────────────────┘             │
│                         ▼                                │
│              (func_id,symbol,direction,signal_time)       │
│              相同 → 自动跳过                              │
└─────────────────────────────────────────────────────────┘
```

### 2.1 主路径职责

- 查 `tb_exit_tracker`，抑制已有活跃追踪的入场信号
- 不做出场检查（兜底路径覆盖）
- Engine 和 calculate 完全解耦：engine 不碰 RSIH 序列

### 2.2 兜底路径职责

- 每次 `_compute` 后扫描 B41/B1B/S41/S1B 列
- 从每根入场 bar 之后找第一个 cross bar
- 生成出场信号追加到 signals 列表
- 无状态，不访问 tracker DB

### 2.3 为何去重安全

兜底路径每次都会算出相同时间点的出场信号（cross bar 不变）。`write_signals` 去重 key 是 `(func_id, symbol, direction, signal_time)`，重复自动跳过。**宁多勿少。**

## 3. DB 表

### 3.1 `tb_exit_tracker`

```sql
CREATE TABLE IF NOT EXISTS tb_exit_tracker (
    id            BIGSERIAL PRIMARY KEY,
    symbol        VARCHAR(10) NOT NULL,
    direction     VARCHAR(6) NOT NULL CHECK (direction IN ('buy','sell')),
    status_label  VARCHAR(10) NOT NULL,
    period        VARCHAR(5) NOT NULL,
    func_name     VARCHAR(50) NOT NULL,
    entry_time    TIMESTAMP NOT NULL,
    entry_price   DOUBLE PRECISION,
    exit_time     TIMESTAMP,
    exit_price    DOUBLE PRECISION,
    status        VARCHAR(10) NOT NULL DEFAULT 'active',
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(symbol, direction, status_label, period, func_name)
);
```

- `status = 'active'` → 活跃追踪，入场信号被 suppress
- `status = 'exited'` → 已出场，为审计保留记录
- UNIQUE 约束确保同 symbol+方向+标签+周期+函数 只有一条记录

### 6.2 `mm2_i5_calculate` 兜底路径

在信号组装完成后、return 之前：

```python
EXIT_RULES = [
    ('B41', 'buy',  75, TH_VAL, 'B41-EXIT'),
    ('B1B', 'buy',  75, TH_VAL, 'B1B-EXIT'),
    ('S41', 'sell', TH_VAL, 75, 'S41-EXIT'),
    ('S1B', 'sell', TH_VAL, 75, 'S1B-EXIT'),
]
for label, direction, buy_exit_th, sell_exit_th, exit_tag in EXIT_RULES:
    entry_series = result.get(label)
    if entry_series is None:
        continue
    entry_bars = entry_series[entry_series].index
    if len(entry_bars) == 0:
        continue
    for bar in entry_bars:
        mask = rsih.index >= bar
        scan = rsih[mask].iloc[1:]
        if len(scan) < 1:
            continue
        th = buy_exit_th if direction == 'buy' else sell_exit_th
        cross = _CROSS(th, scan) if direction == 'buy' else _CROSS(scan, th)
        if not cross.any():
            continue
        xbar = scan[cross].index[0]
        ext = end_ts if is_live else xbar
        edir = 'sell' if direction == 'buy' else 'buy'
        oprice = float(result['open'].at[xbar])
        signals.append({
            'time': ext, 'direction': edir, 'price': oprice,
            'open_price': oprice, 'entry_price': float(result['open'].at[bar]),
            'indicator': exit_tag,
            'buy_status': exit_tag if edir == 'buy' else '',
            'sell_status': exit_tag if edir == 'sell' else '',
        })
```

### 6.3 `FunctionEngine.run()` 主路径

在 `registry.execute()` 返回后、`write_signals` 之前：

```python
# 查活跃追踪，过滤已有持仓的入场信号
from stockpush.services.exit_tracker import get_active, upsert, mark_exited
trackers = get_active(symbol, period, func_info.name)
t_buy = {t['status_label'] for t in trackers if t['direction'] == 'buy'}
t_sell = {t['status_label'] for t in trackers if t['direction'] == 'sell'}

filtered = []
for s in signals:
    if s.get('buy_status') in t_buy or s.get('sell_status') in t_sell:
        continue
    filtered.append(s)
signals = filtered

# 注册新入场（在 write_signals 后，确保 signal 先入库）
new_signals = store.write_signals(func_info.id, symbol, signals)
for s in new_signals:
    for code in ('B41', 'B1B'):
        if code in (s.get('buy_status') or ''):
            upsert(symbol, 'buy', code, period, func_info.name, ...)
    for code in ('S41', 'S1B'):
        if code in (s.get('sell_status') or ''):
            upsert(symbol, 'sell', code, period, func_info.name, ...)

# mark_exited（出场信号已由兜底路径生成，engine 层标记 tracker）
for s in new_signals:
    for code in ('B41-EXIT', 'B1B-EXIT', 'S41-EXIT', 'S1B-EXIT'):
        if code in (s.get('indicator') or ''):
            base = code.replace('-EXIT', '')
            d = 'sell' if base in ('B41', 'B1B') else 'buy'
            mark_exited(symbol, d, base, period, func_info.name, ...)
```

## 7. 时序与写入顺序

```
① calculate() → signals（含兜底出场的 exit）
② engine: 查 tracker → suppress 已有持仓的入场
③ engine: write_signals(signals)  ← 先写库
④ engine: upsert 新入场 tracker   ← 后注册 tracker
⑤ engine: mark_exited 出场 tracker
⑥ engine: push_signal_batch
```

**先写库后改 tracker** 确保 crash 后不会丢信号：signal 入库 → crash → tracker 未注册 → 下次 tick 重新触发 → dedup 跳过 → ok。

## 8. 边界情况

| 场景 | 处理 |
|------|------|
| 服务器当天重启 | tracker DB 恢复 → suppress 正常 → 兜底照样扫 cross |
| 多次重复计算同一天 | write_signals 去重 → 无重复推送 |
| Hermes fill_today 调 calculate | 兜底路径生成 exit → write_signals 写入 → 后续 tick 不受影响 |
| backtest() 调 calculate | 兜底路径也生成 exit，信号列表多几条 → **可接受**（回测信号本来就看全部） |
| 同一 bar 同时有 B41 和 B1B | 各自独立扫描，各自生成 EXIT，write_signals 去重保证不重 |
| 出场信号与普通 sell 信号同方向同时间 | direction='sell' 相同，signal_time 可能碰撞 → 极低概率，去重丢失一条；可接受 |
| entry bar 自己就 cross | `iloc[1:]` 跳过 entry bar，出场的 cross 必须在 entry 之后 |

## 9. 扩展新函数指南

新增自定义函数需要出场追踪时：

1. **calculate 端**：加 `EXIT_RULES` 配置，格式 `(label, direction, exit_th_for_buy, exit_th_for_sell, exit_tag)`
2. **engine 端**：`upsert` 和 `mark_exited` 的入场/出场 status_label 集合加新标签
3. **DB**：不需要改，`tb_exit_tracker` 通用
