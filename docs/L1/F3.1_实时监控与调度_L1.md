> **版本**: V1.0  
> **更新日期**: 2026-07-01

## 1. 功能概述

提供交易日内的周期性调度能力，管理 signal 计算任务的启停与生命周期，支持双交易时段（早盘/午盘）的时间窗口约束、headless 模式的自启动流程、自选股池动态加载，以及非交易时段的 warmup 降级。

## 3. 核心要素定义

### 3.1 关键规则

1. **双交易时段约束**：调度器仅在早盘窗口 `09:25–11:35` 和午盘窗口 `13:00:05–15:05` 内执行 job 函数；超出该范围则跳过执行。
2. **自动停止**：当前时间超过午盘结束时间 `15:05` 后，调度器自动停止并移除 job，进程退出。
3. **Warmup 模式**：非交易时段（午休或收盘后调度器尚未停止的阶段）job 被跳过，不执行数据拉取和信号计算。
4. **Headless 启动流程（systemd timer 模式）**：
   - 检查今日是否为交易日，非交易日直接退出；
   - 推送启动通知至飞书；
   - 检查自选股今日除权除息；
   - 若当前时间晚于 09:30，执行延迟启动数据完整性检查（逐 symbol 逐周期补全缺失 K 线数据）；
   - 执行信号补计算（catchup），覆盖当日 09:25 至当前时间区间的信号遗漏；
   - 注册 SIGTERM 信号处理器，用于优雅关闭；
   - 启动调度器循环，阻塞至收盘或收到 SIGTERM。
5. **自选股池动态加载**：每次 job 执行时通过 `PoolWatcher.get_symbols()` 从 `tb_stock_pool` 实时拉取最新自选股列表，并检测变更。
6. **周期调度**：每个 job 周期（默认 60s）逐一处理每只自选股的 `1m`/`5m`/`30m`/`1d` 四个周期，执行数据下载→下载日志记录→信号计算。
7. **信号补计算（Catchup）**：在延迟启动时，以独立线程执行一次全量信号计算，覆盖开盘至当前时间段的信号遗漏，超时阈值 5 分钟。

## 5. 接口规范（重点）

### 5.1 Export API（本模块提供）

#### RealtimeScheduler（`scheduler.py`）

```python
class RealtimeScheduler:
    def __init__(
        self,
        morning_start: str = "09:25",
        morning_end: str = "11:35",
        afternoon_start: str = "13:00:05",
        afternoon_end: str = "15:05",
        interval_seconds: int = 60,
    ) -> None: ...

    def start(self, job_func, job_id: str = "f51_realtime_job") -> None:
        """启动调度器，按 interval_seconds 周期执行 job_func。
        每次触发时检查是否在交易时段内：
        - 在交易时段内 → 执行 job_func
        - 超过午盘结束时间 → 自动停止调度器
        - 其他时段（午休等）→ 跳过执行
        """

    def stop(self, job_id: str = "f51_realtime_job") -> None:
        """停止调度器，移除 job 并关闭 BackgroundScheduler。"""

    def is_running(self) -> bool:
        """返回调度器是否正在运行。"""

    def get_next_run_time(self, job_id: str = "f51_realtime_job") -> Optional[datetime]:
        """返回 job 的下一次调度执行时间，job 不存在时返回 None。"""
```

## 7. Reset 机制

| 级别 | 操作 | 影响范围 |
|------|------|----------|
| **软重置** | 重启 worker 进程 | 调度器重新初始化，自选股池重新加载，所有运行时状态重置 |
| **硬重置** | 不适用 | 本模块不拥有数据表，无数据重置需求 |

**IMMUTABLE 边界**：`tb_stock_pool` 的数据由 F8.x 模块写入，本模块不执行任何 `INSERT`/`UPDATE`/`DELETE` 操作，仅只读查询。

## 9. 依赖与约束

### 外部依赖

- **APScheduler** (`apscheduler`): `BackgroundScheduler` + `IntervalTrigger`，作为核心调度引擎。
- **XTick** (`xtick`): 交易日历查询和除权除息查询。
- **Feishu Webhook**: 企业飞书机器人消息推送。
- **PostgreSQL**: 通过 `PGConnector` 访问 `tb_stock_pool` 等数据表。

### 内部依赖

- `stockpush/pg_connector.py`: 数据库连接（动态导入）。
- `stockpush/credential_store.py`: 配置解密（仅解密失败时 fallback 原文）。
- `stockpush/services/feishu_pusher.py`: 消息推送。
- `stockpush/services/data_fetcher.py`: 数据获取。
- `stockpush/services/function_registry.py`, `function_engine.py`, `signal_store.py`: 信号计算体系。
- `stockpush/services/download_logger.py`: 下载日志。

### 约束

- 调度器依赖系统时钟；时间跳变可能导致 `_is_within_trading_hours` / `_is_past_end` 判断偏差。
- XTick 交易日历 API 可能因网络问题不可用，此时 fallback 至简单的周末判断（周一至周五均视为交易日，精度较低）。
- `SIGTERM` 处理器仅在 headless 模式下注册；交互式模式下使用 `SIGINT` / `SIGBREAK` 直接退出。
- 调度器使用 `wait=False` 参数关闭，不等待正在执行的 job 完成。

## 10. 验收标准（完成标志）

1. `RealtimeScheduler` 在配置的早盘/午盘窗口内按 `interval_seconds` 周期调用 job_func，窗口外跳过执行。
2. 超过午盘结束时间 `15:05` 后调度器自动停止，`is_running()` 返回 `False`。
3. `worker.py --headless` 在非交易日直接退出，在交易日完整执行启动→检查→补数→计算→调度→收盘的完整流程。
4. 延迟启动时（当前时间 > 09:30），`_check_and_fill_data` 补全缺失 K 线数据并推送通知。
5. 收到 `SIGTERM` 后调度器停止，进程退出前推送停止通知及运行时长。
6. `PoolWatcher` 每次调用 `get_symbols()` 返回 `tb_stock_pool` 的最新数据，变更检测哈希机制正常工作。
7. 信号补计算 catchup 线程在 5 分钟内超时退出不阻塞主流程。
