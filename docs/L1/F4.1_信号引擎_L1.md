> **功能编号**: F4.1  
> **版本**: V1.0  
> **更新日期**: 2026-07-01  
> **对应L0导航表**: F4.1 — 信号引擎（信号函数注册、信号计算、信号归档与推送）

---

## 1. 功能概述

信号引擎是 stockpush 系统的信号计算中枢，负责管理用户自定义信号函数的注册与动态加载、按周期批量执行启用函数、将产生的信号归档至数据库，并通过飞书推送通知用户。业务价值在于将交易策略信号从开发到推送的流程标准化、可配置化，支持多周期、多标的并行信号计算。

## 3. 核心要素定义

### 3.1 关键规则

1. **函数注册中心**：所有可执行的信号函数必须先注册到 `tb_signal_functions` 表，注册记录包含模块路径、函数名、所属周期、参数集 ID 和启用状态。
2. **函数签名契约**：注册的函数必须满足统一签名——
   ```python
   def func_name(symbol: str, period: str, start: str, end: str,
                 param_set_id: int = 0) -> dict
   ```
   返回格式必须包含 `{'signals': [...]}`，其中每个 signal dict 具备 `time`（Timestamp）、`direction`（"buy"/"sell"）、`price`（float）、`open_price`（float）、`indicator`（str）等字段。
3. **去重写入**：`SignalStore.write_signals` 按 `(func_id, symbol, direction, signal_time)` 四元组去重，已存在的记录跳过，不覆盖。
4. **周期匹配**：`FunctionRegistry.get_by_period` 支持 `period` 前缀匹配，例如 `"30m"` 可匹配 `"30m:05"` 等子标签。
5. **只计算不推送（回测模式）**：`FunctionEngine.backtest` 方法执行纯计算，不写入 `tb_signal_log`、不推送飞书。
6. **错误隔离**：`FunctionEngine.run` 中单个函数-标的执行失败不会阻断其他组合的计算，异常被捕获并记日志后继续。

### 3.2 数据表结构

#### tb_signal_functions（模块拥有）

信号函数注册表，每行表示一个已注册的可执行信号函数。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | SERIAL | PRIMARY KEY | 函数 ID |
| name | VARCHAR(64) | NOT NULL, UNIQUE | 函数名称标识（如 `mf30`、`tf05`） |
| display_name | VARCHAR(128) | NOT NULL | 可读展示名称 |
| module_path | VARCHAR(256) | NOT NULL | Python 模块路径（如 `stockpush.userfunc.TF05`） |
| func_name | VARCHAR(64) | NOT NULL | 模块内函数名（如 `tf05_calculate`） |
| period | VARCHAR(16) | NOT NULL | 计算周期（如 `5m`、`30m`，支持冒号子标签如 `30m:05`） |
| param_set_id | INTEGER | DEFAULT 0 | 默认参数集 ID |
| enabled | BOOLEAN | DEFAULT TRUE | 是否启用 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

**索引**：
- UNIQUE(name)

**拥有权**：本模块拥有写入与重置权限。

## 5. 接口规范（重点）

### 5.1 Export API（本模块提供）

#### FunctionRegistry

`FunctionRegistry` 是信号函数的注册中心，提供注册表查询、参数管理、动态加载和执行。

```python
class FunctionRegistry:
    def __init__(self, db: Optional[PGConnector] = None)

    # 查询
    def get_all(self) -> list[FunctionInfo]
    def get_by_period(self, period: str) -> list[FunctionInfo]
    def get_by_name(self, name: str) -> Optional[FunctionInfo]
    def get_enabled(self) -> list[FunctionInfo]

    # 管理
    def register(self, name: str, display_name: str, module_path: str,
                 func_name: str, period: str,
                 param_set_id: int = 0, enabled: bool = True) -> int
    def remove(self, name: str)
    def set_enabled(self, name: str, enabled: bool)

    # 参数
    def get_params(self, func_name: str, set_id: int = 0) -> dict
    def set_params(self, func_name: str, set_id: int = 0,
                   params: dict = None)

    # 验证
    def validate(self, module_path: str, func_name: str) -> dict

    # 执行
    def execute(self, func_name: str, symbol: str, period: str,
                start: str, end: str,
                param_set_id: int = 0) -> dict
```

`FunctionInfo` 数据类定义：

```python
@dataclass
class FunctionInfo:
    id: int
    name: str
    display_name: str
    module_path: str
    func_name: str
    period: str
    param_set_id: int
    enabled: bool
```

**用途说明**：
- `get_all()` / `get_by_period()` / `get_by_name()` / `get_enabled()`：注册表查询，返回 `FunctionInfo` 列表或单条。
- `register()`：向 `tb_signal_functions` 表插入新函数注册记录，返回新记录 ID。
- `remove()`：按名称删除函数注册记录。
- `set_enabled()`：启停函数。
- `get_params()`：按函数名 + 参数集 ID 查询参数，返回 `{param_key: param_value}` 字典。
- `set_params()`：设置参数集，先删旧参数再插新参数（事务语义）。
- `validate()`：验证模块路径对应的函数签名是否包含 `symbol`、`period`、`start`、`end`、`param_set_id` 五个必需参数。
- `execute()`：动态加载模块 + 调用函数，返回函数原始返回值（`{'signals': [...]}`）。

---

#### FunctionEngine

`FunctionEngine` 是信号计算编排层，组合 `FunctionRegistry`、`SignalStore`、`FeishuPusher` 完成"执行-归档-推送"全流程。

```python
class FunctionEngine:
    def __init__(self, registry: FunctionRegistry, store: SignalStore,
                 pusher: FeishuPusher,
                 symbols_getter: Callable[[], list[str]])

    @property
    def registry(self) -> FunctionRegistry

    def run(self, start: str, end: str)
    def backtest(self, func_name: str, symbol: str, period: str,
                 start: str, end: str,
                 param_set_id: int = 0) -> list[dict]
```

**用途说明**：
- `run(start, end)`：遍历所有启用函数，对每个函数的每个标的调用 `registry.execute()`；有信号时调用 `store.write_signals()` 归档 + `pusher.push_signal()` 推送。单个函数-标的错误被捕获并记日志后跳过。
- `backtest()`：回测模式，仅计算返回信号列表，不归档不推送。
- `symbols_getter`：由调用方注入的回调函数，返回当前需计算的标的列表（例如从 `PoolWatcher` 获取）。

#### SignalStore

```python
class SignalStore:
    def __init__(self, db: Optional[PGConnector] = None)

    def write_signals(self, func_id: int, symbol: str,
                      signals: list[dict]) -> list[dict]

    def get_today(self, func_name: Optional[str] = None) -> list[dict]
    def get_history(self, symbol: str,
                    func_name: Optional[str] = None,
                    limit: int = 50) -> list[dict]
```

**用途说明**：
- `write_signals()`：批量写入信号，按 `(func_id, symbol, direction, signal_time)` 四元组去重，返回实际新插入的信号列表。
- `get_today()`：查询当天产生的所有信号，JOIN `tb_signal_functions` 获取 `display_name`，支持按 `func_name` 过滤。
- `get_history()`：按标的查询历史信号，支持按函数名称过滤和数量限制。

class SignalEngine:

                      test_df: pd.DataFrame = None) -> dict

    def is_loaded(self) -> bool
    @property

**用途说明**：
- 旧版单函数引擎，用于单次加载一个信号函数并执行计算。

#### 信号函数契约（用户自定义函数）

所有注册到 FunctionRegistry 的信号函数必须遵循以下签名：

```python
def *_calculate(symbol: str, period: str, start: str, end: str,
                param_set_id: int = 0) -> dict:
    """信号计算入口。

    Args:
        symbol:   股票/合约代码（如 "588200"）
        period:   计算周期（如 "5m", "30m"）
        start:    起始时间 "YYYY-MM-DD HH:MM:SS"
        end:      截止时间 "YYYY-MM-DD HH:MM:SS"
        param_set_id: 参数集 ID，用于从 tb_signal_function_params 获取参数

    Returns:
        {'signals': [
            {'time': Timestamp, 'direction': 'buy'|'sell',
             'price': float, 'open_price': float,
             'indicator': str, 'buy_status': str, 'sell_status': str},
            ...
        ]}
    """
```

当前已注册实现：

| 名称 | 函数 | 周期 | 内部取数来源 |
|------|------|------|------------|
| MF30 | `mf30_calculate` | 30M | `tb_raw_30m` + `tb_raw_1d` |
| MF05 | `mf05_calculate` | 5M | `tb_raw_5m` + `tb_raw_30m` |
| TF05 | `tf05_calculate` | 5M / 30M | `tb_raw_5m` + `tb_raw_30m` + `tb_raw_1d` |
| MM2 | `mm2_calculate` | 1d | `tb_raw_1d` |

## 7. Reset 机制

### 7.1 软重置

- 调用 `FunctionRegistry.set_enabled(name, False)` 可暂时停用某个信号函数，无需删除注册记录。
- 调用 `FunctionRegistry.set_params(func_name, set_id, params)` 可热更新参数集，下次执行时生效。

### 7.2 硬重置

- **`tb_signal_functions`**：`DELETE FROM tb_signal_functions` 可清空所有注册，函数注册记录允许全量重建。
- **`tb_signal_function_params`**：级联删除（`ON DELETE CASCADE`），跟随所属函数的删除。
- **`tb_signal_log`**：允许按条件清理（如 `DELETE FROM tb_signal_log WHERE created_at < '2026-01-01'`）。

### 7.3 IMMUTABLE 边界

- 信号日志 `tb_signal_log` 一旦写入即不可修改（IMMUTABLE），只允许 `INSERT` 和 `DELETE`（清理），不允许 `UPDATE`。这一约束通过应用层确保，不使用数据库触发器。
- `tb_signal_functions` 和 `tb_signal_function_params` 的记录支持通过 `register()`/`remove()`/`set_params()` 管理。

## 8. 异常与日志

### 8.1 异常处理

| 场景 | 异常类型 | 处理方式 |
|------|----------|----------|
| 执行未注册的函数 | `ValueError` | FunctionRegistry.execute 直接抛出 |
| 执行已禁用函数 | `ValueError` | FunctionRegistry.execute 直接抛出 |
| 模块加载失败 | `ImportError` | FunctionRegistry.execute 抛出 |
| 函数未在模块中找到 | `AttributeError` | FunctionRegistry.execute 抛出 |
| 函数执行时内部异常 | `RuntimeError` | FunctionRegistry.execute 包装抛出 |
| `FunctionEngine.run` 中单函数失败 | `Exception` | 捕获记 `logger.exception`，继续下一组合 |
| `write_signals` 写入异常 | `Exception` | 捕获记 `logger.exception`，跳过该批信号推送 |
| `push_signal` 推送异常 | `Exception` | 捕获记 `logger.exception`，继续下一信号 |

### 8.2 日志关键点

| 日志级别 | 场景 |
|----------|------|
| INFO | FunctionEngine.run 开始/结束、每个函数开始执行、无信号跳过、参数更新 |
| WARNING | 跳过无效方向的信号记录 |
| ERROR/EXCEPTION | 函数执行失败、信号写入失败、推送失败、模块加载失败、验证失败 |

## 9. 依赖与约束

### 9.1 系统依赖

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Python | >= 3.10 | 运行时 |
| psycopg (psycopg 3) | >= 3.1.0 | PostgreSQL 数据库连接 |
| pandas | >= 2.0.0 | 信号函数内 DataFrame 计算 |
| numpy | >= 1.24.0 | 数值计算 |

| requests | >= 2.31.0 | 飞书 Webhook HTTP 调用 |
| apscheduler | >= 3.10.0 | 调度器（HermesAPI 使用） |

### 9.2 架构约束

| 约束 | 说明 |
|------|------|
| 单进程执行 | `FunctionEngine.run` 在主线程同步执行，上层调度器决定并发策略 |
| 信号函数无状态 | 每个 `*_calculate` 调用应独立、无副作用（除数据库取数外） |
| 周期隔离 | 不同周期的信号函数在同一轮 `run` 中均执行，上层可根据 `func_info.period` 决定调用时机 |
| 模块缓存 | `FunctionRegistry` 使用 `_module_cache` 缓存已加载的 Python 模块，减少重复导入开销 |
