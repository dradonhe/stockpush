# F5.1.20 系统初始化 功能契约文档（L1）

> 功能编号：F5.1.20 / 版本：V1.0 / 日期：2026-07-01 / 对应 L0 导航表：F5.1.20

## 1. 功能概述

在远端 PostgreSQL 数据库中幂等创建 F5.1 信号系统运行所需的全部 13 张基础表，提供连接校验与表创建能力。作为部署到 VPS 后运行一次即可完成数据库准备的命令行工具。

**与 F51 Web 版（`docs/L1/F1.1_系统初始化_L1.md`）的区别**：

| 对比项 | F5.1.20 stockpush（本模块） | F1.1 F51 Web版 |
|--------|----------------------------|----------------|
| 用途 | 生产 VPS 数据库初始化 | 本地开发环境系统状态检测 |
| 界面 | CLI 命令行 | Flask Web 页面 |
| 数据库 | PostgreSQL（psycopg 3） | — |
| 操作 | 建表 → 可选清理无关表 | 环境检测 → 状态面板 |
| 运行方式 | `python -m stockpush.services.sys_init` | HTTP GET /api/v1/system/status |

## 3. 核心要素定义

### 3.1 关键规则

1. **幂等创建**：所有 `CREATE TABLE IF NOT EXISTS`，可反复执行不报错、不丢数据。
2. **仅 F5.1 核心表**：只创建 `F51_TABLE_SQLS` 中 13 张表，不涉及本地测试表或 Web UI 表。
3. **无关表标记**：`UNRELATED_TABLES`（16 张）仅作清理参考，`drop_unrelated_tables(confirm=True)` 需显式确认才执行。
4. **无数据写入**：只建表，不 INSERT 种子数据或业务数据。
5. **连接自愈**：`PGConnector._ensure_connection()` 每次操作前检查连接，断开自动重连。

### 3.2 数据表结构（重点）

本模块管理 13 张表（定义见 `F51_TABLE_SQLS`），以下列出核心业务表；完整字段定义见 `DATABASE_TABLES_L1.md`。

## 5. 接口规范（重点）

### 5.1 Export API（本模块提供）

```python
class F51SysInit:
    """F5.1 专用数据库初始化器"""

    def check_connection(self) -> bool:
        """验证 PostgreSQL 连接。
        执行 SELECT 1 探活。成功返回 True，失败返回 False 并记录错误日志。
        """

    def init_tables(self) -> Dict[str, Any]:
        """创建 F5.1 所需全部 13 张核心表。
        遍历 F51_TABLE_SQLS，逐条执行 CREATE TABLE IF NOT EXISTS。
        返回 {"ok": [sql_short, ...], "failed": [{"sql": str, "error": str}, ...]}。
        """

    def list_unrelated_tables(self) -> List[str]:
        """返回与 F5.1 无关的表名列表（仅供参考）。
        这些表可在 VPS 上安全手动清理。
        """

    def drop_unrelated_tables(self, confirm: bool = False) -> List[str]:
        """删除与 F5.1 无关的表。
        需要 confirm=True 才实际执行；返回实际删除的表名列表。
        """

    def close(self):
        """关闭数据库连接。连接已关闭或未初始化时安全无操作。"""
```

**CLI 入口**：

```bash
# 初始化全部表
python -m stockpush.services.sys_init

# 输出示例：
# ==================================================
#   F5.1 数据库初始化
# ==================================================
# [INFO] PostgreSQL 连接成功
# [INFO] 表初始化完成: 13 成功, 0 失败
#
# 与 F5.1 无关的表（可在 VPS 上手动清理）:
#   DROP TABLE IF EXISTS tb_alert_log CASCADE;
#   ...
```

## 7. Reset 机制

### 7.1 软重置

重新执行 `init_tables()`：幂等操作，已存在的表不会被重建或清空。

### 7.2 硬重置

不提供。如需清空所有 F5.1 数据，手动执行：

```sql
DROP TABLE IF EXISTS tb_signal_log CASCADE;
DROP TABLE IF EXISTS tb_signal_function_params CASCADE;
DROP TABLE IF EXISTS tb_signal_functions CASCADE;
-- 然后重新运行 python -m stockpush.services.sys_init
```

### 7.3 IMMUTABLE 边界

禁止删除或修改以下数据：
- `F51_TABLE_SQLS` 列表中的任何 DDL（版本控制管理）
- `config/telegram_credentials.key`（Fernet 加密密钥）

## 8. 异常与日志

### 8.1 异常分类

| 级别 | 场景 | 处理方式 |
|------|------|----------|
| ERROR | PostgreSQL 连接失败 | 记录日志 + sys.exit(1) |
| ERROR | SQL 执行失败 | 记录日志 + 在结果集中记录 failed |
| WARNING | 删除无关表未确认 | 记录日志 + 跳过（不执行 DROP） |

### 8.2 日志格式

```python
logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(name)s: %(message)s")
```

日志示例：
```
2026-07-01 10:00:00,000 [INFO] stockpush.services.sys_init: PostgreSQL 连接成功
2026-07-01 10:00:00,100 [INFO] stockpush.services.sys_init: 表初始化完成: 13 成功, 0 失败
```

## 9. 依赖与约束

### 9.1 技术依赖

| 依赖 | 最低版本 | 用途 |
|------|----------|------|
| Python | ≥ 3.10 | 运行时 |
| PostgreSQL | ≥ 14 | 数据库 |
| psycopg | ≥ 3.1 | 数据库驱动 |
| python-dotenv | ≥ 1.0 | 环境变量加载 |
| cryptography | ≥ 41.0 | 凭据加密（credential_store 间接依赖） |

### 9.2 禁止项

- 禁止在 `init_tables()` 之外创建新表（所有表创建必须通过 `F51_TABLE_SQLS` 管理）
- 禁止修改 `F51_TABLE_SQLS` 中已有表的 DDL 而不更新 L1 文档
- 禁止在生产环境自动执行 `drop_unrelated_tables()`（需手动 confirm=True）
