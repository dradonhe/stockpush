## 1. 功能概述

飞书推送模块提供通过飞书 Webhook Bot 向指定群组发送消息的能力，覆盖实时信号推送、系统状态通知、异常告警等功能场景。支持 HMAC SHA256 签名校验，确保消息来源可信。

## 3. 核心要素定义

### 3.1 关键规则

1. **重试策略**：发送失败时最多重试 3 次，仅对飞书限频错误码（`11232`）启用指数退避重试（间隔 2^attempt 秒）；超时与网络异常不重试，直接返回失败。
2. **签名机制**：当 `sign_enabled=True` 且 `sign_secret` 非空时，对每条消息附加时间戳和 HMAC SHA256 签名（`base64(hmac_sha256(timestamp + "\n" + sign_secret))`），飞书服务端校验消息来源。
3. **静默跳过**：`enabled=False` 或 `webhook_url` 未配置时，推送静默跳过，记录 WARNING 日志，不抛出异常。
4. **消息格式**：当前仅支持 `msg_type="text"` 文本消息，`_build_payload()` 内建组装结构。
5. **加密存储**：`webhook_url` 与 `sign_secret` 在 `config.local.yaml` 中以 Fernet 对称加密密文存储，运行时通过 `credential_store.decrypt_value()` 解密。
6. **单例模式**：通过模块级 `_pusher_instance` 全局变量 + `get_pusher()` / `init_pusher()` 提供全局唯一 `FeishuPusher` 实例，避免重复创建。

### 3.2 数据表结构

本模块**无数据表**。

## 6. 配置文件规范

**路径**：`config.yaml` / `config.local.yaml`（`config.local.yaml` 覆盖 `config.yaml`）

**Key 结构**：

```yaml
feishu:
  enabled: true              # 是否启用飞书推送
  sign_enabled: false        # 是否启用 HMAC 签名校验
  sign_secret: ""            # 签名密钥（明文或 Fernet 加密密文）
  webhook: ""                # 飞书 Webhook URL（明文或 Fernet 加密密文）
```

**加密说明**：
- `webhook` 和 `sign_secret` 可通过 `credential_store.encrypt_value()` 加密后存储
- 解密在初始化阶段由调用方（`worker.py` / `hermes_api.py`）统一完成
- 密钥文件位于 `stockpush/config/telegram_credentials.key`，与 Telegram Bot 共用

## 8. 异常与日志

### 8.1 日志

| 日志级别 | 触发条件 | 输出内容 |
|----------|----------|----------|
| INFO | 推送成功 | `Push success: {消息首行前60字符}` |
| WARNING | 推送跳过（未启用或无 webhook） | `Push skipped: enabled={}, webhook={}` |
| WARNING | 推送失败（飞书返回非零 code） | `Push failed: code={} msg={}` |
| WARNING | 推送限频重试 | `Push rate limited (code=11232), retrying in {wait}s (attempt {n}/{3})...` |
| WARNING | 推送超时 | `Push timeout: webhook={前40字符}` |
| ERROR | 网络异常/请求异常 | `Push error: {异常消息}` |

### 8.2 异常处理策略

- `requests.exceptions.Timeout`：捕获后直接返回 `{"success": False, "message": "推送超时"}`，不重试。
- `requests.exceptions.RequestException`：捕获后直接返回 `{"success": False, "message": "推送异常: {原因}"}`，不重试。
- 飞书 API 返回 `code != 0` 或 HTTP status != 200：根据 code 判断，code=11232（限频）时指数退避重试（最多 3 次），其余错误直接返回。
- **本模块不抛出任何异常**，所有错误均通过返回值传递。

## 9. 依赖与约束

### 运行时依赖

| 依赖 | 版本/备注 |
|------|-----------|
| Python | >= 3.8 |
| `requests` | HTTP 客户端 |
| `cryptography` | Fernet 加密（调用方使用，非直接依赖） |

### 外部依赖

| 依赖 | 用途 |
|------|------|
| 飞书 Webhook Bot | HTTP POST 接收端，地址由用户配置 |

### 约束

1. 同一飞书 Webhook 的调用频率上限由飞书平台控制（code=11232 表示到达限频），本模块采用指数退避应对。
2. 消息内容仅支持纯文本格式（`msg_type="text"`），不支持富文本/卡片消息。
3. 签名算法为飞书 Bot 标准 HMAC SHA256 方案，服务端需同步配置签名密钥。
