# F5.2 数据取数 功能契约文档（L1）

> 功能编号：F5.2 / 版本：V1.0 / 日期：2026-07-01
> 对应 L0 导航表行号：F5.2 数据取数

---

## 1. 功能概述

从 primary 数据源获取 K 线行情数据并进行标准化后存入 PostgreSQL 数据库，为信号计算模块（F4.x）提供统一的 raw 数据层。支持 1 分钟 / 5 分钟 / 30 分钟 / 日线四个周期，采用 INSERT ... ON CONFLICT DO UPDATE 幂等写入，不覆盖历史数据。

## 3. 核心要素定义

### 3.1 关键规则

1. **主数据源机制**：通过 `datasources.primary` 配置指定主数据源（默认 `xtick`）；当主数据源不可用时，通过 `_get_active_provider()` 自动检测降级，遍历注册表中的 Provider。
2. **分钟级数据分月拉取**：1m / 5m / 30m 数据按自然月分段请求（`_iter_month_ranges`），避免单次请求行数超限。
3. **幂等写入**：使用 `INSERT ... ON CONFLICT (ts, symbol) DO UPDATE`，重复写入不产生重复行。
4. **Intraday 种子填充**：5m/30m 盘中无实时数据时，用今日日线 open 值垫入第一根合成 K 线（时间戳对齐实际交易开始时间：5m→09:35, 30m→10:00），后续真实数据通过 ON CONFLICT DO UPDATE 自动覆写。
5. **资产类型检测**：通过代码前缀和数据库查询自动区分股票/基金，影响复权处理。
6. **标准化适配层**：`StandardAPIAdapter` 定义 12 个标准 API 接口（API01–API12），所有 Provider 通过继承实现标准化数据访问。
7. **数据完整性检查**：`check_today_data_complete()` 查询当日是否有数据记录，用于判断是否需要补拉。

### 3.2 数据表结构（重点）

#### K 线数据表（F5.2 拥有）

四张 K 线表结构完全一致，仅表名不同：

**`tb_raw_1m`** — 1 分钟 K 线
**`tb_raw_5m`** — 5 分钟 K 线
**`tb_raw_30m`** — 30 分钟 K 线
**`tb_raw_1d`** — 日线 K 线

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| ts | TIMESTAMP | NOT NULL | K 线时间戳 |
| symbol | VARCHAR(10) | NOT NULL | 股票/基金代码 |
| open | DECIMAL(10,3) | NOT NULL | 开盘价 |
| high | DECIMAL(10,3) | NOT NULL | 最高价 |
| low | DECIMAL(10,3) | NOT NULL | 最低价 |
| close | DECIMAL(10,3) | NOT NULL | 收盘价 |
| vol | BIGINT | NOT NULL | 成交量（股数） |
| amount | DECIMAL(20,2) | NOT NULL | 成交金额（元） |

**约束**：
- PRIMARY KEY (ts, symbol)

**拥有权**：F5.2 拥有写入与重置权限。其他模块只读访问。

## 5. 接口规范（重点）

### 5.1 Export API（本模块提供）

#### `DataFetcher.__init__`

```python
def __init__(
    self,
    primary: str = "xtick",
    db_path: str = None
) -> None
```

初始化数据抓取器。延迟初始化 SrcMgrService（F5.1）和 PGConnector。

- `primary` — 主数据源名称，对应 `config.yaml` 中 `datasources.primary`
- `db_path` — 数据库路径（可选，默认使用项目配置）

#### `DataFetcher.fetch_and_save`

```python
def fetch_and_save(
    self,
    symbol: str,
    period: str = '1d',
    adjust: str = 'qfq',
    log_callback=None
) -> Tuple[bool, int, str]
```

拉取当日 K 线数据并保存到数据库。

- `symbol` — 股票/基金代码
- `period` — 周期：`'1m'` / `'5m'` / `'30m'` / `'1d'`
- `adjust` — 复权类型：`'qfq'`（前复权）/ `'hfq'`（后复权）/ `'None'`
- `log_callback` — 可选回调，传入原始 DataFrame 用于日志记录
- **返回** `(success, saved_count, source)`：
  - `success` — bool，是否成功保存至少一条记录
  - `saved_count` — int，实际写入数据库的记录数
  - `source` — str，实际使用的数据源名称

**特殊行为**：5m/30m 周期无数据时，自动尝试用今日日线 open 垫入合成 K 线（`_try_seed_intraday`）。

#### `DataFetcher.fetch_and_save_history`

```python
def fetch_and_save_history(
    self,
    symbol: str,
    period: str = "1d",
    start: str = None,
    end: str = None,
    adjust: str = "qfq"
) -> Tuple[bool, int, str]
```

拉取历史 K 线数据并保存到数据库。

- `symbol` — 股票/基金代码
- `period` — 周期：`'1m'` / `'5m'` / `'30m'` / `'1d'`
- `start` — 起始日期 `YYYY-MM-DD`（默认一年前）
- `end` — 结束日期 `YYYY-MM-DD`（默认今日）
- `adjust` — 复权类型
- **返回** `(success, saved_count, source)`

#### `DataFetcher.fetch_kline`

```python
def fetch_kline(
    self,
    symbol: str,
    period: str = '1d',
    adjust: str = 'qfq'
) -> Optional[pd.DataFrame]
```

获取当日 K 线数据（仅今日），不做持久化。

- **返回** DataFrame（列：ts, open, high, low, close, vol, amount）或 None

#### `DataFetcher.fetch_history`

```python
def fetch_history(
    self,
    symbol: str,
    period: str = "1d",
    start: str = None,
    end: str = None,
    adjust: str = "qfq"
) -> Optional[pd.DataFrame]
```

获取历史 K 线数据，不做持久化。1m/5m/30m 分钟级数据自动按自然月分段请求；1d 日线一次请求。

- `start` — 起始日期（默认一年前）
- `end` — 结束日期（默认今日）
- **返回** DataFrame 或 None

#### `DataFetcher.save_to_db`

```python
def save_to_db(
    self,
    data: List[Dict[str, Any]],
    period: str = '1d'
) -> int
```

将标准化后的 K 线数据列表写入数据库。通过 `DataDownService._normalize_kline_row` 进行时间戳标准化后，使用 `INSERT ... ON CONFLICT DO UPDATE` 写入对应周期表。

- `data` — 原始数据字典列表
- `period` — 周期
- **返回** 成功写入的记录数

#### `DataFetcher.check_today_data_complete`

```python
def check_today_data_complete(
    self,
    symbol: str,
    period: str = '1d'
) -> bool
```

检查当日 K 线数据是否已存在于数据库。

- **返回** True（有数据）/ False（无数据或查询失败时默认返回 True）

#### `DataFetcher.get_current_source`

```python
def get_current_source(self) -> str
```

获取最近一次成功请求所使用的数据源名称。无数据时返回空字符串。

#### `DataFetcher._ensure_table`

```python
def _ensure_table(self, period: str) -> str
```

确保周期对应的数据表存在，不存在则自动创建。返回表名。表结构见 3.2 节。

#### `DataFetcher._detect_asset_type`

```python
def _detect_asset_type(self, symbol: str) -> str
```

根据代码判断资产类型：优先从 `tb_stock_pool` 获取用户自定义类型，其次通过代码前缀判断交易所基金（ETF/LOF），最后查询 `tb_fund_codes` 表。

- **返回** `'stock'` 或 `'fund'`

#### Provider 通用接口（`BaseProvider` / `StandardAPIAdapter`）

每个 Provider 通过继承 `StandardAPIAdapter` 实现标准化接口，核心 K 线接口：

```python
def fetch_kline(
    code: str,
    period: str,
    start: str,
    end: str,
    adjust: str = 'qfq',
    asset_type: str = 'stock'
) -> Union[pd.DataFrame, int]
```

- 成功返回 DataFrame，失败返回 -1

## 7. Reset 机制

### 硬重置操作

K 线数据表的硬重置即 DROP + 重建表：

```sql
DROP TABLE IF EXISTS tb_raw_1m;
DROP TABLE IF EXISTS tb_raw_5m;
DROP TABLE IF EXISTS tb_raw_30m;
DROP TABLE IF EXISTS tb_raw_1d;
```

重建在下次调用 `_ensure_table()` 时自动完成。

### 硬重置范围

F5.2 拥有如下表：
- `tb_raw_1m` — 可重置
- `tb_raw_5m` — 可重置
- `tb_raw_30m` — 可重置
- `tb_raw_1d` — 可重置

### 软重置操作

删除指定时间范围内的数据：

```sql
DELETE FROM tb_raw_<period> WHERE symbol = ? AND ts >= ? AND ts <= ?;
```

### IMMUTABLE 数据边界

F5.2 无不可变数据边界。历史数据可被重新获取和覆写（UPSERT 语义），但正常流程下不主动删除已保存数据。

## 8. 异常与日志

### 日志命名

使用 `logging.getLogger(__name__)` 获取 logger，日志名称为 `stockpush.services.data_fetcher`。

### 日志场景

| 场景 | 级别 | 说明 |
|------|------|------|
| SrcMgrService 初始化失败 | WARNING | 不影响构造，Provider 降级使用 |
| Provider fetch_kline 失败 | WARNING | 打印 provider 类名和异常信息 |
| 时间戳标准化失败 | WARNING | 使用原始数据写入 |
| 单条记录插入失败 | WARNING | 打印表名和异常，继续下一条 |
| save_to_db 整体失败 | ERROR | 全量写入失败 |
| Intraday 种子填充 | INFO | 记录垫入的周期、代码、价格 |
| Intraday 种子填充失败 | WARNING | 不影响主流程 |
| check_today_data_complete 异常 | WARNING | 默认返回 True |

### 异常处理策略

- 所有 Provider 调用异常由 `_fetch_range()` 捕获并返回 None，不传播到上层
- Provider 实例化失败返回 None，不影响其他 Provider
- 数据库连接异常在调用点捕获，逐层返回默认值（False / 0 / True）

## 9. 依赖与约束

### 强依赖

| 依赖 | 版本/范围 | 说明 |
|------|-----------|------|
| Python | >= 3.10 | 运行时 |
| psycopg | >= 3.1 | PostgreSQL 驱动 |
| pandas | >= 2.0 | 数据帧处理 |

### 软依赖

| 依赖 | 说明 |
|------|------|
| 各 Provider 包 | 数据源 Provider 的依赖 Python 包，未安装时对应 Provider 不可用 |
| PostgreSQL | 数据存储后端，无 PG 时无法持久化 |

### 约束

- 同一 `(ts, symbol)` 主键约束确保幂等写入
- 分钟级数据（1m/5m/30m）每次请求限制在一个自然月内
- 外部 Provider API 的速率限制由各 Provider 自行控制

## 10. 验收标准（完成标志）

1. 可通过 `DataFetcher(primary="xtick").fetch_and_save("000001", "1d")` 拉取股票日线数据并写入 `tb_raw_1d` 表
2. 重复调用 `fetch_and_save` 不会产生重复行（主键幂等性验证）
3. `fetch_history("000001", "1d", "2026-01-01", "2026-06-30")` 返回指定日期范围的完整 DataFrame
4. `PERIOD_TABLE_MAP` 中 4 个周期均可在调用 `_ensure_table` 时自动建表
5. 主数据源不可用时，通过 `_get_active_provider()` 降级到其他可用 Provider
6. `check_today_data_complete("000001", "1d")` 在有数据时返回 True，无数据时返回 False
7. 资产类型检测：ETF 代码（如 `510050`）正确识别为 `'fund'`，普通股票代码识别为 `'stock'`
8. 5m/30m 盘中无数据时自动垫入合成 K 线（`_try_seed_intraday`）
9. 配置文件 `datasources.primary` 变更后，新构造的 `DataFetcher` 生效
